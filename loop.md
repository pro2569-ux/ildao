# /loop — ildao 자동 작업 루프

매 반복마다 **미완료 작업 하나**를 끝까지 진행하고, **반드시 실제 명령으로 검증**한다.
범위를 넘기지 않는다. 한 반복 = 한 항목.

## 1. 작업 선택 (하나만)
다음 우선순위로 **자동 진행이 안전한** 항목 중 가장 높은 것 하나만 고른다:
1. **깨진 검증 복구가 최우선** — 아래 검증 명령 중 실패하는 게 있으면 그것부터.
2. `docs/TODO.md`의 `[자동가능]` 미완료 항목 중 최상위.
3. 대화에서 끝내지 못한 작업.

**자동 진행 금지(건너뛴다)**: `docs/TODO.md`의 `[사용자필요]` 항목 — Firebase 배포(자격증명), 외부 서비스(FCM·Sentry·GA4·App Check), 의존성/아키텍처 결정(TanStack Query·SSR 등). 사용자가 직접 지시할 때만 한다.

## 2. 구현 (그 항목만)
고른 항목 하나만 구현한다. 곁다리 리팩터·다른 항목 손대기 금지.

## 3. 검증 (반드시 실제로 실행)
이 프로젝트의 실제 검증 명령을 **직접 실행**해 출력을 확인한다:
- `npx tsc --noEmit` — 타입 검사 (`npm run typecheck` 스크립트가 생기면 그걸로)
- `npm run build` — Next 빌드(컴파일 + 타입 검사). **현재의 실질 게이트**
- `npx vitest run` — **Vitest가 설정된 경우에만**. 아직 테스트 러너 없음 → 없으면 생략
- 린트: `next lint`는 미설정이라 대화형 프롬프트로 멈춤 → 설정 전까지 **실행하지 않는다**

> ⚠️ 검증 명령을 **실제로 실행하지 않고 통과를 추정하지 마라.** 출력을 직접 읽는다.

## 4. 판정
- **실패** → 출력을 읽고 원인을 수정한 뒤 **다시 검증**. 통과할 때까지 같은 항목을 반복한다.
- **통과** → 변경 내용을 `docs/TODO.md`에 한 줄로 기록하고 그 항목을 `[x]`로 완료 처리한 뒤, 논리 단위로 커밋한다. 진행 메모리(`improvement-campaign.md`)에도 반영.

## 5. 종료
`[자동가능]` 미완료 항목이 없고 모든 검증이 통과하면 **"전부 완료 — 검증 통과"** 한 줄로만 보고하고, 다음 작업을 스케줄하지 않는다(루프 종료). 종료 직전 `PushNotification`으로 한 줄 결과를 보낸다.

## 규칙
- 검증 명령을 실제 실행하지 않고 통과를 추정하지 않는다.
- 테스트를 통과시키려고 테스트를 **약화/삭제/skip 처리하지 않는다**. (테스트가 진실, 코드가 틀린 것이다.)
- 한 반복에 한 항목. 범위 초과 금지.
- 커밋: 원격 없음 → `main`에 직접. `git commit -m` **인라인만**(에디터 띄우면 세션 멈춤). 메시지에 개선 번호 명시, 푸터 `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- 모든 변경은 **되돌릴 수 있는 로컬 작업**이어야 한다. 비가역/외부 작업(배포·삭제·전송)은 사용자 승인 전까지 하지 않는다.
- 작업트리가 더러우면(미커밋 변경) 먼저 맥락을 파악하고, 내가 만든 변경이면 검증 후 커밋해 정리한다.

## 배경 (이 프로젝트의 현실)
- 결함 51건은 전건 완료(`docs/defect-report.md`). 남은 건 개선 42건(`docs/improvement-report.md`) — 백로그는 `docs/TODO.md`.
- 원격/PR/CI 없음. Firestore 룰/인덱스는 **미배포** 상태(코드 수정과 별개로 사용자가 `firebase login` 후 배포해야 실효) — 루프는 배포하지 않는다.
- 검증 명령은 점진적으로 현실화된다: typecheck 스크립트(#316 일부)를 추가하면 `npm run typecheck`가, Vitest(#328)를 도입하면 `npx vitest run`이 검증에 합류한다.
