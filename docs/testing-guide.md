> 작성일: 2026-06-19 · 최신 커밋: `5e83447`

## 0. 한 줄 요약
코드는 완성도 높음(결함 전건 + 개선 다수 수정, 프로덕션 빌드 통과). **단, 아직 어디에도 배포돼 있지 않아 공개 웹 주소가 없습니다.** 확인하려면 내 PC에서 개발 서버를 띄워 **http://localhost:3000** 으로 봐야 합니다.

## 1. 현재 상황

### 코드
- 결함 리포트(`docs/defect-report.md`) 49건 전건 처리, 개선 항목 다수 적용. `main` 브랜치에 커밋, 작업트리 깨끗, `npm run verify`(typecheck+build) 통과 → 앱은 정상 컴파일/빌드됨.

### 배포 (← 여기가 핵심)
| 대상 | 상태 |
|---|---|
| 공개 웹사이트(Vercel 등) | ❌ **배포 안 됨** — URL 없음 |
| GitHub 원격 | ❌ 미연결 (로컬 git만) |
| Firestore 보안 룰 / 인덱스 | ⚠️ **미배포** (코드엔 작성·커밋됨, Firebase엔 반영 안 됨) |

### 환경변수 (`.env.local`)
| 키 | 상태 |
|---|---|
| Firebase 6종 (API_KEY·AUTH_DOMAIN·PROJECT_ID·STORAGE_BUCKET·MESSAGING_SENDER_ID·APP_ID) | ✅ 코드와 이름 일치, 설정됨 |
| Kakao 지도 키 | ⚠️ **이름 불일치** — 코드는 `NEXT_PUBLIC_KAKAO_MAP_KEY`, 파일엔 `NEXT_PUBLIC_KAKAO_MAP_API_KEY` → 지도 미표시(주소 텍스트만) |
- Firebase 프로젝트: `ildao-fcbf6`

## 2. 웹에서 볼 수 있나요? 주소는?
- **공개 주소는 없습니다.** (배포 전)
- **로컬 개발 서버**로만 볼 수 있습니다 → **http://localhost:3000**
- 단순히 화면(UI)만 보는 건 dev 서버만으로 가능하지만, **실제 동작(로그인→공고→지원)** 검증은 아래 0단계(Firestore 배포)가 선행돼야 안전합니다.

## 3. 테스트 절차

### 0단계 (권장 선행) — Firestore 룰/인덱스 배포
배포 안 하면 일부 기능이 `permission-denied` / `failed-precondition`(인덱스 없음)으로 실패할 수 있습니다(특히 **지역 필터**, **지원하기**).
```bash
npx firebase-tools login            # 브라우저 인증 (세션에선 ! 프리픽스로 실행)
npx firebase-tools deploy --only firestore   # 룰 + 인덱스 배포 (프로젝트 ildao-fcbf6)
```
> 빠르게 화면만 볼 거면 이 단계는 건너뛰어도 되지만, 데이터 읽기/쓰기가 깨질 수 있음을 감안하세요.

### 1단계 — 로컬 실행
```bash
npm run dev
```
→ 브라우저에서 **http://localhost:3000** 접속. (Google 로그인은 localhost 기본 허용이라 동작)

### 2단계 — 역할별 클릭 체크리스트
한 번에 다 보려면 **구인자 계정 1개 + 구직자 계정 2개**(Google 계정)로 교차 테스트하면 좋습니다.

**구직자 흐름**
- [ ] Google 로그인 → 회원가입(역할: 구직자, 전화번호 형식 검증 동작 확인)
- [ ] 홈(WorkerHome) → 구인공고 피드(`/jobs`)
- [ ] **지역 필터**(주황 칩) + 직종 필터 + 정렬 동작 *(지역 필터는 인덱스 배포 필요)*
- [ ] 공고 상세 → **지원하기 + 한 줄 메시지** / **즐겨찾기(공고·업체)** 토글
- [ ] 지원 내역(`/my-applications`) → 수락된 건에서 **업체 전화 버튼** 노출
- [ ] 공수 계산기(`/calculator`) 개인용: 날짜 클릭 → **바텀시트 모달**, 음수 입력 차단, 휴무 토글
- [ ] 즐겨찾기(`/favorites`) 3개 탭

**구인자 흐름**
- [ ] 회원가입(역할: 구인자) → 홈(EmployerHome) 대시보드 통계
- [ ] 구인글 작성(`/jobs/create`): **지역(시/도) 선택**, 일당/인원/길이 검증
- [ ] 내 구인글(`/my-jobs`) → **지원자 N명 보기** → **지원자 관리** 화면
- [ ] 지원자 **수락/거절** → 수락 시 **연락처(전화)** 노출
- [ ] 구직자 검색(`/workers`): 직종 필터에 **'기타' 포함** 확인, 즐겨찾기 토글
- [ ] 공수 계산기 팀장용: 팀원 추가, 팀원 바텀시트 모달

## 4. 알려진 제약 / 주의
1. **Firestore 미배포** → 0단계 안 하면 지역필터·지원 등이 에러날 수 있음. (특히 이번에 추가한 지역 복합 인덱스 4종, 지원 스냅샷 룰)
2. **Kakao 지도** → env 변수명 불일치로 현재 **지도 대신 주소 텍스트만** 표시. 고치려면 `.env.local`의 `NEXT_PUBLIC_KAKAO_MAP_API_KEY` → `NEXT_PUBLIC_KAKAO_MAP_KEY`로 이름 변경 후 dev 재시작.
3. **PWA**는 개발 모드에서 비활성(`next.config.js`에서 dev disable) → 설치/오프라인은 `npm run build && npm run start`로 확인.
4. **테스트 데이터가 없음** → 처음엔 공고/지원이 비어 있으니, 구인자로 공고 몇 개 올린 뒤 구직자로 지원해야 흐름이 보입니다.
